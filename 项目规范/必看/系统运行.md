## 执行环境规范

### 仓库目标识别（SYS-011）

在执行 git、备份、门禁、测试、系统修改等操作前，必须先识别当前操作仓库：

| 仓库类型 | 路径 | 职责 | 状态 |
|----------|------|------|------|
| **根仓库** | `D:/汇度编辑部1/写作知识库` | 项目规范、sources、projects、交接文档 | 活跃 |
| **V2 系统目录** | `medical_kb_system_v2/` | 当前生产系统代码 | **默认入口** |
| **V1 独立子仓库** | `medical_kb_system/` | 旧版系统代码 | 只读/归档计划 |

**识别命令**：
```powershell
# 获取当前目录所属仓库
git rev-parse --show-toplevel
```

**操作规则**：
- 修改 `项目规范/` 内容 → 在根仓库操作
- 修改 V2 系统 → 在根仓库操作（V2 由根仓库管理）
- V1 已进入只读状态，不应修改

### Windows PowerShell 命令统一

Windows 环境下必须输出 PowerShell 可执行命令：

| 禁止语法 | 正确语法 |
|----------|----------|
| `export VAR=value` | `$env:VAR = "value"` |
| `VAR=value command` | `$env:VAR = "value"; command` |
| `<<EOF ... EOF` | 使用 `-File` 或 `-Command` 参数 |
| `&&` 链式命令 | `;` 或分多行 |
| `2>/dev/null` | `2>$null` 或 `| Out-Null` |
| `rm -rf` | `Remove-Item -Recurse -Force` |

**文件路径**：使用反斜杠 `\` 或在 PowerShell 中使用正斜杠 `/` 均可。

### Git 认证策略

**原则**：远程写操作允许交互认证，不强制非交互模式。

| 场景 | 策略 |
|------|------|
| 私有远程 push/pull | 允许 `GIT_TERMINAL_PROMPT=1`，走交互认证 |
| CI/自动化环境 | 使用 SSH 密钥或 token，无需交互 |
| 认证失败 | 切换到交互认证，不重复尝试非交互模式 |

### 失败回退机制

当操作失败时，应识别失败类型并切换策略，而不是原样重复：

| 失败类型 | 回退策略 |
|----------|----------|
| Shell 语法不兼容 | 切换到 PowerShell 语法 |
| 认证失败 | 切换到交互认证 |
| 历史污染阻塞 push | 使用 `--force-with-lease` 或干净副本上传 |
| 命令不存在 | 检查 PATH 或使用完整路径 |
| 权限不足 | 提示用户以管理员身份运行 |

**回退原则**：
- 连续 2 次相同命令失败 → 必须切换策略
- 策略切换后仍失败 → 暂停并报告错误详情

## AI协作最小门禁

### 单阶段单权威（SYS-012）

- 每个 `Phase` 只能有一个正式执行入口。
- 启动 brief 仅用于生成并启动正式计划文件。
- 正式计划登记到总控后，后续施工只认该计划文件，不再同时服从 brief。
- 总控、进度表、brief、正式计划口径冲突时，以总控已登记的正式计划为唯一执行真相源。

### 状态切换原子更新（SYS-013）

- 阶段或任务从 `待启动 / 进行中` 切到 `已验证 / 已关闭` 时，必须在同一轮同步更新：
  - 总控
  - 进度追踪表
  - 当前阶段正式计划
- 未同步完成前，不得对外宣称该阶段已验证或已关闭。

### 完成口径三级定义（SYS-014）

- `已实现`：文件、代码、索引、输出已生成。
- `已验证`：已实现，且已通过文件核对、测试、命令、解析或目录复验。
- `已关闭`：已验证，且真相源同步、验收条件满足、遗留项已明确处理。
- 任何文档、报告、完成说明都不得跳级宣称。

### 结构化真相源门禁（SYS-015）

- 对 `JSON / manifest / registry / batch 清单` 类文件，必须至少校验：
  - 语法可解析
  - 关键字段存在
  - 数字统计自洽
  - 不存在重复键、旧问题残留、已删除对象残留、或与目录实物冲突的声明
- `pytest` 通过仅证明代码链可运行，不替代真相源校验。

### 批量输出与统计安全（SYS-016）

- 批量生成文件前，必须先做目标文件名合法性检查。
- Windows 保留名 `CON / PRN / AUX / NUL / COM1-9 / LPT1-9` 不得作为输出或中间文件名。
- 批次统计必须使用固定字段并保持含义稳定：
  - `source_count`
  - `processed_count`
  - `excluded_count`
  - `output_count`
  - `asset_count`
- 若 scope 发生变化，必须在同一轮同步更新 `included / excluded / 分项统计 / 总计统计`。

## 项目日志机制

> **目标**: 避免所有过程文档集中堆积在主系统日志目录，确保每个独立项目都有自己的完整过程留痕
> **适用范围**: 所有写作任务、审核任务、风格预览、过程说明、问题记录

### 双层日志机制

系统采用"双层日志"保存策略，**现为代码自动能力**：

1. **主系统日志**
   - 作用：作为全项目总索引，记录本次任务做了什么、属于哪个项目、完整日志放在哪里
   - 位置：`D:\汇度编辑部1\写作知识库\写作项目运行日志\`
   - 内容要求：只保留摘要索引，不保存完整过程文档
   - **自动触发**：`Application.run()` 完成交付时自动写入

2. **项目内日志**
   - 作用：保存某个独立项目的完整过程文档
   - 位置：`D:\汇度编辑部1\写作知识库\projects\{项目名}\项目日志\`
   - 内容要求：保存问题记录、关键决策、执行摘要、风格预览、中间稿说明等完整过程材料
   - **自动触发**：项目任务 (`task_category="project"` 且 `project_name` 已设置) 完成时自动写入

### 任务分类判断

**代码级判断**（通过 `RouteContext` 配置）：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `task_category` | 任务类别 | `"project"` |
| `project_name` | 项目名称 | `None` |
| `deliverable_type` | 交付物类型 | `None` |

**判断逻辑**（`RouteContext.is_project_task()`）：
- `task_category == "project"` 且 `project_name is not None` → 独立项目任务
- 其他情况 → 系统治理任务

### 项目写作任务启动与变更

- 独立项目写作任务启动前，必须先基于 [06_项目写作任务模板.md](../06_项目写作任务模板.md) 创建项目专用执行卡。
- 医学内容项目在此基础上，还必须同时读取 [07_医学项目写作规范.md](../07_医学项目写作规范.md)。
- 交付契约、结构方案、证据范围一旦确认，即视为冻结约束。
- 后续冲突需求不得直接覆盖前序冻结约束，必须先写入执行卡中的 `变更记录` 段。
- 变更记录形成后，实际变更条目还必须同步写入项目日志，作为时间线留痕。

### 日志内容

**主系统摘要日志**（Markdown 格式）：
- 任务 ID、创建时间、完成时间
- 产品 ID、语体等级、受众
- 项目名称、交付物类型
- 输出路径、状态、产物列表
- 项目详细日志位置（仅项目任务）

**项目内详细日志**（Markdown 格式）：
- 完整任务信息表格
- 执行结果、产物列表
- 完整摘要 JSON
- 扩展元数据

### 使用示例

```python
from engine.runtime import Application
from engine.contracts import RouteContext

# 项目任务 - 自动双层日志
context = RouteContext(
    product_id="lecanemab",
    register="R3",
    audience="专科医师",
    project_name="仑卡奈单抗患者教育项目",
    deliverable_type="article",
    task_category="project"  # 默认值，可省略
)

app = Application()
result = app.run(context)
# 自动写入：
# - 写作项目运行日志/{timestamp}_{项目名}_{task_id}.md
# - projects/仑卡奈单抗患者教育项目/项目日志/{timestamp}_{task_id}_detail.md

# 系统治理任务 - 只写主系统日志
system_context = RouteContext(
    product_id="system",
    register="R1",
    task_category="system"
)
result = app.run(system_context)
# 只写入：写作项目运行日志/{timestamp}_{task_id}.md
```
