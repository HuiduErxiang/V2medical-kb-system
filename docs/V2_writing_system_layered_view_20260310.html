V2 写作系统 · 压缩理解版

底层还是 L1 到 L4，只是在外面包了两层壳

如果你觉得现在的 V2 比最开始复杂很多，这个感觉是对的。但真正新增的，不是知识分层本身，而是“知识怎么被稳定地用来生产文章”。 所以最容易理解的方式，不是推翻原来的四层，而是把它看成：底层还是 L1 到 L4，外面再包一层文章处理流程壳，再包一层项目治理和留档壳。

最早的四层

它回答的是“系统知道什么”。也就是写作规则、医学规则、疾病知识、产品知识分别放在哪里。

现在新增的壳

它回答的是“系统怎么稳定写文章”。包括任务进入方式、证据选择、写作规划、交付留档。

你可以这样记

L1 到 L4 没被推翻，只是从“知识库”升级成了“知识库加生产流程”。

三层理解法

如果把 V2 压缩回最容易理解的样子，它可以被看成三层：最底下是知识四层，中间是文章处理主链，最外面是项目治理和留档。

最外层：项目治理与留档壳

这一层不直接写内容，但它负责把项目规则固定住，并把过程留痕。它处理的是：项目执行卡、交付契约冻结、结构冻结、变更记录、项目日志和主系统日志。

项目启动 冻结约束 变更记录 双层留档

中间层：文章处理流程壳

这一层负责真正把知识拿来用。它处理的是：任务定义、拿证据、定论点、排大纲、生成写作指令、过基础检查、形成交付结果。

任务定义 证据选择 写作规划 交付

最底层：L1 到 L4 知识底盘

这还是原来的四层，只是现在被系统真正消费了。换句话说，L1 到 L4 不再只是“静态放在仓库里”，而是成为文章处理主链的底盘。

L1

写作技法、表达习惯、行文原则。现在主要落在规则层里。

L2

医学写作 playbook、面向不同受众和语气的策略。

L3

疾病层知识。定义疾病、机制、终点、场景等较稳定知识。

L4

产品层知识。定义具体产品、证据、定位、表达资产。

所以最适合你的理解方式不是“V2 变成了一个完全不同的怪物”，而是“原来的 L1 到 L4 还在，只是外面多了流程壳和治理壳”。

L1 到 L4 在 V2 里分别落到了哪里

最开始你脑子里的四层，现在没有消失，而是被拆散后放进了几个更可运行的位置。下面这张对照表，就是“原来的层”和“现在系统里的落点”的对应关系。

L1：写作技法层

现在主要落在规则资产里，对应的是写作 craft、结构习惯、表达原则。它本质上还是在，但从“一个孤立层级”变成了可被文章主链直接消费的规则资源。

rules/l1_writing_craft

L2：医学写作 playbook 层

现在也主要落在规则资产里，但更偏写作策略、受众适配、医学表达方式。它负责告诉系统“面对不同读者、不同语气，文章应该怎么讲”。

rules/l2_medical_playbook persona_profiles

L3：疾病知识层

现在主要落在结构化知识区。它不直接变成文章句子，而是先成为可查询、可调用的疾病知识底盘，再被文章处理流程拿来组织论点。

structured/l3

L4：产品知识层

现在落在结构化产品知识和重建后的证据资产里。它既负责产品层信息，也负责更接近证据和定位的内容。

structured/l4 evidence/rebuilt

为什么现在看起来不是纯四层了

因为 V2 不再只问“知识放哪”，还问“知识怎么被文章消费”。所以原来的四层继续存在，但被放进了更接近运行的资产区。

你可以这样记忆

  • L1/L2 更偏规则和写法。
  • L3/L4 更偏知识和证据。
  • 文章主链站在它们上面做编排。

一篇文章在这三层里怎么走

如果把“文章是怎么被产出来的”压缩回这个三层模型，它其实就是：先在最外层把规则定住，再在中间层跑写作流程，底下不断调用 L1 到 L4 的知识底盘。

第一步：在最外层，把规则定住

先确认这篇文章的项目归属、交付契约、引用要求、结构方向和后续变更入口。这一步是治理层，不是正文层。

第二步：在中间层，拿证据和知识

系统开始去调用底盘：从 L1/L2 拿写法和策略，从 L3/L4 拿疾病和产品知识，从证据层拿事实点和片段，然后组合成当前这篇文章真正可用的材料。

第三步：在中间层，生成写作方案

它会把这些材料整理成：核心论点、大纲、写作角度、关键证据和风格要求。这个阶段仍然不是最终正文，而是“可执行写法”。

第四步：交付后回到最外层，自动留档

文章交付后，系统再把这次任务写回项目日志和主系统摘要。这样以后出现问题时，不是只剩正文，而是能顺着项目规则、证据选择和交付结果往回查。

为什么你会觉得它复杂了很多

因为最开始的四层架构，只负责回答“系统知道什么”。现在的 V2 还要回答“系统怎么稳定写文章”，所以复杂度是被新增工作带上去的，不是原来的四层坏掉了。

最开始的四层架构

  • 重点：知识怎么分层摆放
  • 解决：规则、疾病知识、产品知识分别放哪里
  • 优势：简单、清楚、容易理解
  • 不足：没把“项目怎么推进、文章怎么落地、规则怎么留痕”纳进来

现在的 V2

  • 重点：知识怎么被稳定地拿来生成文章
  • 新增：任务定义、证据选择、写作规划、交付、留档
  • 优势：更稳、更可追溯、更适合多项目协作
  • 代价：看起来不再像一张简单的 L1 到 L4 图
所以现在的复杂，不是“系统失控”，而是“系统从知识库进化成了生产系统”。只要你把它重新压回“三层理解法”，就不会觉得它乱。
