# 规章制度

> **文档位置**: `project_standards/规章制度.md`
> **性质**: 静态规范文档，铁律唯一权威来源
> **更新触发**: 规则变更时更新（需同步所有引用文档）
> **最后更新**: 2026-03-08

---

## 命名空间

规则采用命名空间前缀区分用途：

- **SYS-xxx** — 系统运行铁律（系统级约束）
- **DOC-xxx** — 文档维护铁律（文档操作约束）
- **可扩展命名空间**: DATA-xxx（数据治理）、FLOW-xxx（流程）、QA-xxx（质量）

## 第一部分：系统运行铁律（SYS-001 ~ SYS-011）

> **违反后果**: 可能导致数据污染、系统故障或内容不合规
> **使用场景**: 每次新会话开始时，AI必须复述全部约束规则确认已读

| 编号 | 铁律 | 说明 |
|------|------|------|
| **SYS-001** | **L3 铁律** | 疾病知识不能含特定产品名。判断标准：换药还能用=属于 L3 |
| **SYS-002** | **隔离铁律** | 禁止跨产品 L3/L4 污染。每个产品的证据库必须独立 |
| **SYS-003** | **Qwen 禁令** | Qwen 有中文路径 bug，禁止碰文件。文件读写由 flash(quick)代劳 |
| **SYS-004** | **文献引用规范** | 必须读正文提取完整学术引文，不能用路径名/文件名引用 |
| **SYS-005** | **先确认需求** | 启动对话后必须先向用户索要 SPEC需求文档 |
| **SYS-006** | **JSON key不动** | 用户明确推迟JSON key语义化改造，保持现有命名 |
| **SYS-007** | **L3 纯度门例外** | sources/citation/citation_zh字段允许出现产品名（引用需要） |
| **SYS-008** | **交付格式** | 文章必须直接写入 word，不得用.md格式，也不得由 md格式转为 word |
| **SYS-009** | **署名过滤** | 多作者公众号文章按正式署名过滤，非署名内容不入库 |
| **SYS-010** | **保存规则** | 当用户说"结束任务"时，更新此文件，禁止新建文档 |
| **SYS-011** | **仓库目标识别** | 所有 git、备份、门禁、测试、系统修改动作执行前，必须先识别当前操作仓库。根仓库：`D:/汇度编辑部1/写作知识库`（project_standards、sources、projects）；核心系统仓库：`medical_kb_system/`（核心系统代码、引擎、测试、知识库）。禁止在错误仓库内提交、推送或修改。 |
## 第二部分：文档维护铁律（DOC-001 ~ DOC-021）

> **违反后果**: 可能导致文档不一致、维护困难、版本混乱
> **使用场景**: 文档操作（创建、更新、归档）时必须遵守

| 编号 | 铁律 | 说明 |
|------|------|------|
| **DOC-001** | **先读规范** | 每次新对话必须先完整阅读本文档 |
| **DOC-002** | **必读清单** | 按顺序阅读 01_项目日志 → 02_版本迭代 → 03_当前状态 → 04_待办事项 |
| **DOC-003** | **保存规则** | 用户说"结束任务"时更新对应文档，禁止新建文档 |
| **DOC-004** | **单一更新** | 正文原地修改，更新记录追加新条目；涉及版本号、日期、规则总数、测试状态、当前阶段等共享状态时，必须遵循唯一权威源（03_当前状态.md、系统运行.md、04_待办事项.md），其他文档仅做同步，不得各自独立定义。 |
| **DOC-005** | **版本递增** | 每次更新必须递增 Phase 版本号 |
| **DOC-006** | **日期同步** | 更新后同步修改顶部"日期"和文末"最后更新" |
| **DOC-007** | **格式纯净** | 禁止表格行首添加前缀字符（如`WQ||`、`#PK|`） |
| **DOC-008** | **去重检查** | 更新后检查无重复段落 |
| **DOC-009** | **隔离铁律** | L3 疾病知识不能含特定产品名；禁止跨产品 L3/L4 污染 |
| **DOC-010** | **文献引用规范** | 必须读正文提取完整学术引文，不能用路径名/文件名引用 |
| **DOC-011** | **先确认需求** | 启动对话后必须先向用户索要 SPEC需求文档 |
| **DOC-012** | **直接写 Word** | 文章必须直接写入 word，不得用.md格式，也不得由 md格式转为 word |
| **DOC-013** | **JSON key不动** | 用户明确推迟JSON key语义化改造，保持现有命名 |
| **DOC-014** | **上下文保护** | 对话上下文>220K tokens 时自动总结、清空、重载 |
| **DOC-015** | **落盘卫生门** | 所有文档写入前必须通过文本卫生检查：禁止行首异常前缀、表格结构破坏、重复段落或重复 checklist 行、非预期控制标记；任一项失败即阻断正式保存，先修复后再执行"结束任务"。 |
| **DOC-016** | **模型异常防护** | 若检测到模型输出异常（如文本污染、重复写入、格式错乱），立即暂停写入，隔离异常内容，改用备用模型重新生成，确认无误后再落盘。 |
| **DOC-017** | **项目日志分层保存** | 凡能明确归属到某个独立项目的过程文档，必须优先保存到该项目目录下的 `项目日志/`；主系统日志目录仅保留摘要索引，记录任务概况与项目内完整日志路径。禁止将独立项目的完整过程文档长期集中存放于主系统日志目录，导致项目目录空心化。 |
| **DOC-018** | **结束任务先分类** | 执行"结束任务"前，必须先判断本次任务属于"系统治理任务"还是"独立项目任务"。只有系统治理任务才允许更新 `project_standards/01_项目日志.md`、`02_版本迭代.md`、`03_当前状态.md`、`04_待办事项.md` 及相关交接文档；独立项目任务不得误触发系统治理文档更新。无法判断时，默认按独立项目任务处理，禁止先更新系统文档后再回退。 |
| **DOC-019** | **技术操作安全原则** | 修改核心文档时，必须保持 UTF-8 编码一致性，采用可回溯的整块替换方式，禁止使用不明确编码、批量盲替换、直接拼接追加等高风险操作。保存后必须通过文本卫生检查、结构完整性检查和重复段落检查，未通过不得落盘。 |
| **DOC-020** | **禁止删除式修复** | 禁止通过删除原文件后重建的方式修复核心文档。若需修复，必须使用原地编辑或整块替换方式保留原文件路径和创建历史。 |
| **DOC-021** | **结束任务一致性校验** | 执行"结束任务"时，凡涉及系统治理任务，必须完成动态文档一致性校验。`03_当前状态.md`、`04_待办事项.md`、`必看/系统运行.md`、`必看/交接文档.md`、`02_版本迭代.md` 之间的版本号、日期、测试状态、规则总数、已完成事项口径必须一致；任一文档未同步、残留失效待办或交付声明与实际文件不符，均不得视为任务完成。 |

## 引用方式

其他文档引用规则时，只需写规则编号：

- 正确：`遵守 SYS-001 规则` → 索引到本文档查看完整定义
- 错误：`遵守 L3 铁律（疾病知识不能含特定产品名）` → 全文重复
