# Known Issue: GLM-5 输出延迟错位

## 问题描述

GLM-5 模型在完成任务后，存在输出延迟发送的问题，导致：

1. 任务完成后的总结输出没有立即发送
2. 用户发送新消息后，**旧的输出被发送**，新的请求被忽略
3. 用户不得不重复发送请求

## 复现场景

```
时间线：
17:33:22 → [assistant] 完成 README，但总结输出 pending
17:33:49 → [user] 发送 "帮我实现 Python 函数..."
17:33:49 → [assistant] 8ms 后输出了 README 总结（答非所问）
17:34:16 → [user] 再次发送 Python 函数请求
17:34:16 → [assistant] 才开始处理 Python 函数任务
```

## 根因分析

- 任务完成后的输出没有立即 flush
- 输出缓存在队列中，等待下一个触发时机
- 新消息到达时，**旧输出被发送**，新请求未处理

## 影响范围

- 用户感知：AI 答非所问、忽略指令
- 需要用户重复发送请求
- 影响交互体验和任务连续性

## 临时解决方案

1. 用户：如果 AI 答非所问，重复发送请求
2. AI 行为改进：
   - 每个操作完成后立即确认，不延迟输出
   - 使用 `todowrite` 让状态可见
   - 任务结束显式说「✅ 完成」
